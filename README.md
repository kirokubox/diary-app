# Yuki Diary App

自分専用のWeb日記アプリです。

日々の出来事・感情・思考を自由に書きながら、メタ認知、言語化、文章力を少しずつ育てるためのアプリです。Google Keepのように長文を自由に書ける体験を残しつつ、検索、整理、バックアップ、復元をしやすくしています。

現在はPWA対応済みです。Pixel 8などのAndroid Chromeで開き、ホーム画面に追加すると、普通のスマホアプリに近い感覚で起動できます。

## 開発環境での起動方法

初回だけ、依存パッケージをインストールします。

```bash
npm install
```

アプリを起動します。

```bash
npm run dev
```

起動すると、ターミナルに次のようなURLが表示されます。

```bash
http://localhost:5173/
```

このURLをPCのChromeなどで開くと、日記アプリを使えます。

## スマホから確認する方法

開発中のアプリをPixel 8などのスマホから確認する場合は、PCとスマホを同じWi-Fiに接続してください。

PC側で次のコマンドを実行します。

```bash
npm.cmd run dev -- --host 0.0.0.0
```

ターミナルに `Network` のURLが表示されます。

```bash
Network: http://192.168.1.10:5173/
```

この `Network` URLをPixel 8のChromeで開くと、スマホから開発中のアプリを確認できます。

注意: `http://192.168...` のようなLAN内URLは、スマホでの画面表示確認には使えます。ただし、Service Workerは原則としてHTTPSまたはlocalhostで有効になるため、LAN内HTTPではPWAのインストールやオフライン動作を完全には確認できない場合があります。PWAとして本格確認する場合は、HTTPSで配信するか、実際の運用URLで確認してください。

## ホーム画面に追加する方法

Pixel 8 / Android Chromeでの手順です。

1. Pixel 8のChromeでアプリURLを開く
2. 右上のメニューを開く
3. 「ホーム画面に追加」または「アプリをインストール」を選ぶ
4. ホーム画面に追加されたアイコンから起動できることを確認する

PWAとして起動すると、ブラウザのアドレスバーがない `standalone` 表示に近い形で開けます。

## PWA対応内容

- Web App Manifestを追加
- アプリ名を `Yuki Diary` に設定
- 短い名前を `Diary` に設定
- 表示モードを `standalone` に設定
- 起動URLを `/` に設定
- portrait向けに設定
- 192x192 / 512x512 の仮アイコンを追加
- Service Workerを追加
- 一度開いたあと、最低限オフラインでも画面を開けるようにアプリ本体をキャッシュ

日記データの保存先は、これまで通りブラウザ内のIndexedDBです。PWA化してもクラウド同期はされません。

## デプロイ候補

将来的な公開先の候補は以下です。

- Netlify
- Vercel
- GitHub Pages

どこにデプロイしても、日記本文がサーバー側に保存されるわけではありません。公開されるのはアプリ本体のHTML、CSS、JavaScript、manifest、アイコン、Service Workerです。

ただし、公開URLを知っている人はアプリ画面を開ける可能性があります。自分専用で使う場合は、公開URLをSNSやnoteに載せないでください。

検索避けとして `robots.txt` と `noindex` を入れていますが、これは絶対的なアクセス制限ではありません。本当に他人に見られたくない場合は、URLを共有しない運用が大切です。

## 保存場所とプライバシー注意点

このアプリはサーバーを使いません。ログイン機能も外部データベースもありません。

日記データは、使っているブラウザのIndexedDBに保存されます。つまり、基本的には「その端末の、そのブラウザの中」に保存されます。

注意してください:

- 日記データはブラウザ内保存であり、クラウド保存ではありません。
- スマホとPCでデータは自動同期されません。
- スマホで書いた日記はスマホ側のブラウザに保存されます。
- PCで書いた日記はPC側のブラウザに保存されます。
- Chromeの閲覧データやサイトデータを削除すると、日記データが消える可能性があります。
- 別のブラウザで開くと、同じ日記データは表示されません。
- シークレットウィンドウで使うと、データが残らない可能性があります。
- JSONバックアップをGitHubや公開フォルダに入れないでください。
- 公開URLを不用意に共有しないでください。

本番運用時は、スマホをメイン端末にするか、PCをメイン端末にするかを決めて使うのがおすすめです。複数端末で使う場合は、JSONエクスポートとJSONインポートで手動移行してください。

## バックアップ方法

日記データはブラウザ内に保存されます。大事な日記を失わないため、定期的にJSONバックアップを取ってください。

手順:

1. アプリを開く
2. 下部タブの「設定」を開く
3. 「JSONエクスポート」を押す
4. `diary-backup-YYYY-MM-DD.json` のようなファイルが保存される
5. そのJSONファイルをOneDrive、外付けストレージ、別フォルダなどに保管する

JSONファイルは復元用です。日記データ、テンプレート、タグ候補が含まれます。

本番運用する場合は、週1回以上のJSONバックアップを推奨します。日記をたくさん書く時期は、もっと短い間隔でバックアップしてもよいです。

## 復元方法

JSONバックアップから復元する場合は、設定画面の「JSONインポート」を使います。

手順:

1. アプリを開く
2. 下部タブの「設定」を開く
3. 「JSONインポート」でバックアップJSONを選ぶ
4. 同じ日付の日記がある場合は、上書き確認が表示される
5. 問題なければ復元する

同じ日付の日記は、日付をIDとして扱います。そのため、同じ日付のデータを取り込む場合は上書き対象になります。

## JSONとMarkdownの使い分け

JSON:

- 復元用
- バックアップ用
- 日記データを安全に戻したいときに使う

Markdown:

- 読み返し用
- 共有用
- ChatGPTに貼る用
- 文章として保管したいときに使う

Markdownは見やすい文章形式ですが、復元用ではありません。復元したい場合は、必ずJSONエクスポートを使ってください。

## 基本機能

- 今日の日記を書く
- 日付を変えて過去日記・未来日記を書く
- 体力、気分、タグ、本文を記録する
- 入力後、約1秒で自動保存する
- 保存ボタンで手動保存する
- 一覧で日記を見返す
- キーワード、タグ、日付、気分、体力で検索する
- その日の日記をMarkdownでコピーする
- 全日記をJSONでバックアップする
- 全日記をMarkdownで出力する
- JSONバックアップから復元する
- テンプレートを編集する
- 日記や全データを確認つきで削除する
